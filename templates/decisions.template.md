# Decision Log

> The accountability layer. Every `/ask-the-board` session logs a row here.
> Advice is cheap; this file is where you find out if it was any *good*.
> Copy this to `decisions.md` at the repo root (gitignored — it's personal).

## How it works
1. When the board gives a recommendation, add a row (the skill does this for you).
2. Set a **Review date** — when the outcome will actually be knowable.
3. On/after that date, fill in **Outcome** and a **Verdict**.
4. Periodically ask: *"review my decision log"* — the board reads its own track
   record and learns which advisor's read tends to be right for you.

**Verdict values:** ✅ Right call · ❌ Wrong call · 🤷 Mixed · ⏳ Too early
